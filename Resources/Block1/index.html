The Block1 Case Study

This is the index for the Block-1 case study. This documents the Block-1 language and the components that make up a compiler for the language. These components were developed incrementally according to the recursive-descent development methodology. The compiler does not include code generation.

The compiler is built using CompilerKit a framework for developing object-oriented recursive-descent compilers.

  • The language and example programs
  • main.cpp: The code for the compiler's main(), which can invoke the various version of the parser. The program can be used to invoke three demos:
    • block1 count-let <input_file>: count let statements in a program.
    • block1 parse-pure <input_file>: invoke the pure parser.
    • block1 parse-sema <input_file>: invoke the full parser & semantic analyser.
  • Block1Parser.hpp/cpp: the scanner, based on the Finite State Machine.
  • Block1PureParser.hpp/cpp: the pure parser.
  • Block1Parser.hpp/cpp: the parser with semantics integration.
  • Block1Sema.hpp/cpp: the semantic analyser.
